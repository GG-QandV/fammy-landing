Параллельность: Как прод и разработка живут одновременно
Вариант: Git Branch (рекомендую)
main (прод)                    landing-v3 (разработка)
  │                               │
  │  ← деплоится на fammy.pet     │  ← НЕ деплоится никуда
  │                               │
  │  коммит: "fix: promo input"   │  коммит: "feat: accordion component"
  │  коммит: "style: button text" │  коммит: "feat: pet selector modal"
  │                               │  коммит: "feat: f2 tool page"
  │                               │
  │          ← никаких нахлёстов →│
  │                               │
Как это работает:

Создаём ветку landing-v3 от main
Вся разработка новой архитектуры — только в landing-v3
Багфиксы прода — в main как обычно
Пуши landing-v3 уходят в GitHub, но сервер деплоит только main (в настройках Coolify/Vercel указана ветка main)
На проде ничего лишнего не появляется — сервер даже не знает что ветка landing-v3 существует
Слияние (Cutover)
Когда новая архитектура готова и протестирована:

bash
# Финальная проверка
git checkout landing-v3
npm run build        # всё собирается?
# Да → мержим
git checkout main
git merge landing-v3 --no-ff -m "feat: launch v3 architecture"
git push origin main
# → сервер деплоит новый main → на проде новый лендинг
Откат после слияния
Да, безболезненный откат возможен. Потому что --no-ff создаёт один merge-коммит:

main:  A → B → C → M (merge) → ...
                    ↑
              один коммит, внутри — все изменения v3
Откат — одна команда:

bash
git revert M          # создаёт коммит, отменяющий все изменения из merge
git push origin main  # → сервер деплоит → прод возвращается к старому лендингу
Все коммиты сохраняются в истории. Ничего не теряется. Можно снова смержить после исправлений.

Если нужно продолжить фиксы прода во время разработки v3
bash
# Фикс в проде
git checkout main
# ... делаешь фикс ...
git commit -m "fix: something"
git push origin main
# Подтягиваешь фикс в ветку разработки
git checkout landing-v3
git merge main         # ← забираем фикс из прода в v3
Так ветки не расходятся и при финальном слиянии не будет конфликтов.

Итого
Вопрос	Ответ
Лишнее грузится на прод?	Нет, деплоится только main
Можно пушить в обе ветки?	Да, независимо
Нахлёсты при пушах?	Нет, разные ветки
Откат после слияния?	Одна команда git revert
История коммитов сохраняется?	Да, полностью
